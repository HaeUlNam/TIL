### REST API
- Representational State Transfer라는 약자로 웹(HTTP)의 장점을 최대한 활용할 수 있는 아키텍처입니다.

### REST API 구성
1) 자원: URI
2) 행위: HTTP METHOD
3) 표현(Representations)


### REST API 특징 
- 
- 캐싱 가능: REST API의 큰 특징 중 하나가 HTTP라는 웹 표준을 사용하기에 HTTP의 캐싱 사용 가능
- Client-Server 구조: Client와 Server부분이 개발할 것이 명확하다.
